# Auth0 Management API v2 — Reference

All requests require a valid `access_token` obtained from the token endpoint.
Variables use hurl syntax: `{{variable_name}}`.

---

## Authentication

Both flows return the same response shape. Capture the token the same way regardless of which flow you use:

```bash
ACCESS_TOKEN=$(hurl --variables-file "$ENV_FILE" <token-template>.hurl | jq -r '.access_token')
```

Response: `{ "access_token": "...", "token_type": "Bearer", "expires_in": 86400 }`

---

### Flow 1 — client_credentials (client_id + client_secret)

Use when you have a `client_id` and `client_secret`. Template: `get-token.hurl`.

```hurl
POST {{tenant}}/oauth/token
Content-Type: application/x-www-form-urlencoded
[Options]
cacert: {{cacert}}

grant_type=client_credentials&client_id={{client_id}}&client_secret={{client_secret}}&audience={{resource_server_identifier}}
```

Capture:
```bash
ACCESS_TOKEN=$(hurl --variables-file "$ENV_FILE" "$SKILLS_DIR/templates/get-token.hurl" | jq -r '.access_token')
```

---

### Flow 2 — private_key_jwt (client assertion)

Use when authenticating with a signed JWT instead of a secret. Template: `get-token-assertion.hurl`.

The `client_assertion` is a signed JWT generated by the caller. The `audience` for this flow is the tenant URL itself (e.g. `https://auth.example.com/`), not the Management API audience URL.

```hurl
POST {{tenant}}/oauth/token
Content-Type: application/x-www-form-urlencoded
[Options]
cacert: {{cacert}}

grant_type=client_credentials&client_assertion_type=urn:ietf:params:oauth:client-assertion-type:jwt-bearer&client_assertion={{client_assertion}}&audience={{resource_server_identifier}}
```

Capture:
```bash
ACCESS_TOKEN=$(hurl --variables-file "$ENV_FILE" "$SKILLS_DIR/templates/get-token-assertion.hurl" | jq -r '.access_token')
```

## Clients

### List clients (non-interactive, paginated)

```hurl
GET {{tenant}}/api/v2/clients?per_page=50&page=0&include_totals=true&app_type=non_interactive&include_fields=true&fields=client_id,name,app_type,client_authentication_methods
Authorization: Bearer {{access_token}}
Content-Type: application/json
[Options]
cacert: {{cacert}}
```

Common query parameters:
- `per_page` — results per page (max 100)
- `page` — zero-indexed page number
- `include_totals=true` — adds `total`, `start`, `limit` to response
- `fields` — comma-separated list of fields to include
- `app_type` — filter by type: `non_interactive`, `regular_web`, `native`, `spa`

### Get client by ID

```hurl
GET {{tenant}}/api/v2/clients/{{client_id_target}}
Authorization: Bearer {{access_token}}
Content-Type: application/json
[Options]
cacert: {{cacert}}
```

### Create client

Body file example:
```json
{
  "name": "78N0010227-laila",
  "app_type": "non_interactive",
  "grant_types": ["client_credentials"],
  "jwt_configuration": { "alg": "RS256" },
  "client_metadata": {
    "contractNo": "78N0010227"
  },
  "client_authentication_methods": {
    "private_key_jwt": {
      "credentials": [
        {
          "name": "credential-01",
          "credential_type": "public_key",
          "pem": "-----BEGIN PUBLIC KEY-----\n...\n-----END PUBLIC KEY-----\n",
          "alg": "RS256"
        }
      ]
    }
  }
}
```

```hurl
POST {{tenant}}/api/v2/clients
Authorization: Bearer {{access_token}}
Content-Type: application/json
[Options]
cacert: {{cacert}}
file, {{body_file}};
```

Returns `201` with the full client object on success.

### Update client (metadata, authentication methods, etc.)

Body file example — update metadata only:
```json
{
  "client_metadata": {
    "contractNo": "78N0000702"
  }
}
```

Body file example — replace credential reference on client:
```json
{
  "client_authentication_methods": {
    "private_key_jwt": {
      "credentials": [
        { "id": "cred_fUCepHr6kwpwQz5soPvGi2" }
      ]
    }
  }
}
```

```hurl
PATCH {{tenant}}/api/v2/clients/{{client_id_target}}
Authorization: Bearer {{access_token}}
Content-Type: application/json
[Options]
cacert: {{cacert}}
file, {{body_file}};
```

### Delete client

```hurl
DELETE {{tenant}}/api/v2/clients/{{client_id_target}}
Authorization: Bearer {{access_token}}
[Options]
cacert: {{cacert}}
```

Returns `204` on success.

## Client Credentials

### List credentials for a client

```hurl
GET {{tenant}}/api/v2/clients/{{client_id_target}}/credentials
Authorization: Bearer {{access_token}}
Accept: application/json
[Options]
cacert: {{cacert}}
```

### Add credential to client

Body file example — public key:
```json
{
  "name": "credential-name-02",
  "credential_type": "public_key",
  "pem": "-----BEGIN PUBLIC KEY-----\n...\n-----END PUBLIC KEY-----\n",
  "alg": "RS256"
}
```

Body file example — certificate (PEM):
```json
{
  "name": "credential-cert-01",
  "credential_type": "public_key",
  "pem": "-----BEGIN CERTIFICATE-----\n...\n-----END CERTIFICATE-----\n",
  "alg": "RS256"
}
```

```hurl
POST {{tenant}}/api/v2/clients/{{client_id_target}}/credentials
Authorization: Bearer {{access_token}}
Content-Type: application/json
[Options]
cacert: {{cacert}}
file, {{body_file}};
```

Returns `201` with the new credential object (includes `id` field needed for future operations).

### Update credential (e.g. clear expiry)

Body file example:
```json
{
  "expires_at": null
}
```

```hurl
PATCH {{tenant}}/api/v2/clients/{{client_id_target}}/credentials/{{credential_id}}

[Options]
cacert: {{cacert}}
file, {{body_file}};
```

### Delete credential

```hurl
DELETE {{tenant}}/api/v2/clients/{{client_id_target}}/credentials/{{credential_id}}
Authorization: Bearer {{access_token}}
[Options]
cacert: {{cacert}}
```

## Client Grants

### Get grants for a client

```hurl
GET {{tenant}}/api/v2/client-grants?per_page=50&page=0&include_totals=true&client_id={{client_id_target}}
Authorization: Bearer {{access_token}}
Content-Type: application/json
[Options]
cacert: {{cacert}}
```

Response shape:
```json
{
  "total": 1,
  "start": 0,
  "limit": 50,
  "client-grants": [
    {
      "id": "<grant-id>",
      "client_id": "<client-id>",
      "audience": "<resource-server-identifier>",
      "scope": ["<scope-1>", "<scope-2>"]
    }
  ]
}
```

### Create client grant

Body file example:
```json
{
  "client_id": "<client-id>",
  "audience": "<resource-server-identifier>",
  "scope": [
    "read:resource",
    "create:resource",
    "delete:resource"
  ]
}
```

```hurl
POST {{tenant}}/api/v2/client-grants
Authorization: Bearer {{access_token}}
Content-Type: application/json
[Options]
cacert: {{cacert}}
file, {{body_file}};
```

Returns `201`. Returns `409` if a grant already exists for that client+audience combination.

### Update client grant (replace scope)

The `scope` array is replaced entirely — include all scopes you want active.

Body file example:
```json
{
  "scope": [
    "read:resource",
    "create:resource",
    "update:resource"
  ]
}
```

```hurl
PATCH {{tenant}}/api/v2/client-grants/{{grant_id}}
Authorization: Bearer {{access_token}}
Content-Type: application/json
[Options]
cacert: {{cacert}}
file, {{body_file}};
```

### Delete client grant

```hurl
DELETE {{tenant}}/api/v2/client-grants/{{grant_id}}
Authorization: Bearer {{access_token}}
[Options]
cacert: {{cacert}}
```

## Error Reference

| Status | Meaning |
|---|---|
| `400` | Bad request — malformed body or invalid field values |
| `401` | Unauthorized — missing or expired access token |
| `403` | Forbidden — token lacks required scope |
| `404` | Not found — resource ID does not exist in this tenant |
| `409` | Conflict — resource already exists (e.g. duplicate client grant) |
| `429` | Rate limited — back off and retry |

## jq Filters

```bash
# Extract client_id and name from list response
| jq '[.[] | {client_id, name}]'

# Get the grant ID for a specific audience
| jq '.["client-grants"][] | select(.audience == "<resource-server-identifier>") | .id'

# Extract credential IDs and names
| jq '[.[] | {id, name, credential_type}]'

# Get scopes from a specific grant
| jq '.["client-grants"][0].scope'
```
